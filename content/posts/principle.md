너무 방황하고 있었다. 내가 뭘 해야할지 잘 몰랐다.
1년의 휴식을 했지만 생각보다 내가 한건 없었다.
산업기능요원을 하자니 내 능력은 처참했고, 공부를 하고 있지만 맞는 방향인지 모르겠다.

그러다가 [이 글](https://velog.io/@teo/ai-and-developer)을 봤다.
내가 고민했던 부분도 어느정도 있지만 막연히 공부하고 있던 나에게 뭔가 인사이트를 제시해주는 것 같았다. 많은 영감을 얻었다.
그래서 이제 막연히 공부하기 보다는 어느 정도 AI를 사용하면서 체계적으로 공부해야 할 것 같다.

## 결론적으로 내가 길러야 할 개발자로서의 능력
- 소프트 스킬, 기술적 기반 위에서 자신의 맥락을 담아 구체적인 이야기를 구조적으로 할 수 있는 능력

## 나만의 원칙
- 질문할때, 다른 개발자한테 묻는다고 생각하고, 알아들을 수 있게끔 질문하려고 노력하기
- AI로 구글 검색을 대체
- 추상적인 요구사항보다 구체적인 명시로
  - NO "로그인 기능 만들어줘"
  - YES "Node.js Express로 이메일/비밀번호 입력받아서 bcrypt로 암호화 검증하고, JWT 토큰 발급하는 POST /api/login API 만들어줘. 로그인 실패 시 401 에러 리턴하고, 성공 시 토큰이랑 사용자 정보 리턴해줘"
- 기술문서 작성하기
  - 참고 : https://developers.google.com/tech-writing
- 혼자서 코드 리뷰하기


- AI에게 내가 세운 가설의 약점을 찾게 하기
  - ex) "나는 로그인을 위해 DB에 ID와 PW를 저장하고, 서버 메모리에 로그인 여부를 기록할 거야. 이 방식의 결함(Flaw) 3가지와, 실제 현업에서는 이 결함을 해결하기 위해 어떤 설계 패턴을 쓰는지 알려줘."

- AI에게 정답을 묻지 말고 선택지를 달라고 하기
  - ex) "이 기능을 구현할 때 '가장 쉬운 방법'과 '가장 확장성 있는 방법' 두 가지 아키텍처를 비교해줘. 각각 어떤 라이브러리가 필요하고, 개발 기간은 얼마나 차이 날까?"

- 이런 느낌으로 질문: "나는 [기능 이름]을 만들 건데, 내가 생각한 흐름은 [A -> B -> C]야. 이 방식의 아키텍처적 한계점 2가지를 지적해주고, 이를 보완하기 위한 가장 표준적인 계층 구조(Layered Architecture)를 설계해줘. 코드는 아직 필요 없어."
